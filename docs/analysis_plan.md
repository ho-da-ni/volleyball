# 프로배구 관람 활성화를 위한 지역별 잠재수요 분석 계획서

## 1. 분석 개요

본 분석은 프로배구 관람 데이터와 지역별 인구·체육시설 데이터를 결합하여 전국 17개 시도 단위의 배구 관람 잠재수요를 산출한다. 지역별 관람 수요와 경기 공급의 불균형을 파악하고, 팬 확장 가능성이 높은 지역을 도출하는 것이 핵심 목적이다.

분석 결과는 배구 경기 개최, 지역 이벤트, 팝업 응원전, 유소년 프로그램 운영 우선지역을 제안하는 데 활용한다. 이를 통해 기존 연고지 중심의 관람 활성화 전략을 보완하고, 신규 팬 발굴 및 지역 스포츠 문화 확산을 위한 데이터 기반 의사결정 근거를 마련한다.

## 2. 분석 배경 및 목적

프로배구 관람 수요는 특정 연고지와 인기 구단을 중심으로 형성되는 경향이 있다. 이에 따라 경기 개최 지역에 따라 관람 기회와 팬덤 형성 수준에 차이가 발생한다. 그러나 홈구장이 없거나 경기 공급이 적은 지역에도 배구에 관심을 가질 수 있는 잠재 관람층은 존재할 수 있다.

기존 관람 활성화 전략은 실제 관람객과 기존 연고지 중심으로 수립되는 경우가 많아 신규 팬을 발굴할 수 있는 지역을 데이터 기반으로 파악하는 데 한계가 있다. 따라서 지역별 관람 현황뿐 아니라 인구 구조, 체육시설 인프라, 접근성, 경기 공급 수준을 함께 분석할 필요가 있다.

본 분석의 목적은 다음과 같다.

- 지역별 프로배구 관람 수요와 경기 공급 현황을 비교한다.
- 인구 규모, 청년층·학생층 비중, 체육시설 인프라를 활용해 잠재 관람 기반을 측정한다.
- 수요와 인프라는 높지만 경기 공급이 부족한 지역을 식별한다.
- 지역별 특성에 맞는 경기·이벤트·유소년 프로그램 운영 우선순위를 제안한다.

## 3. 분석 데이터

| 데이터 | 주요 변수 | 활용 목적 |
| --- | --- | --- |
| 프로스포츠 관람 데이터 | 지역, 종목, 시즌, 구단, 경기장, 경기 수, 관람객 수 | 배구 관람 수요와 경기 공급 수준 산출 |
| 전국체육시설표준데이터 | 지역, 시설 유형, 실내외 구분, 가능 종목 | 배구 가능 실내체육시설 기준 인프라 산출 |
| 공공체육시설 보급현황 | 지역, 시설 수, 면적, 수용 규모 | 공공 체육 기반 보완 지표 산출 |

분석 단위는 전국 17개 시도로 통일한다. 데이터별 지역명 표기가 다를 수 있으므로 `서울`, `서울특별시`와 같은 명칭을 표준 행정구역명으로 정규화한다.


### 3.1 현재 반영한 공식 데이터

| 파일 | 기준 | 비고 |

| `data/official/population.csv` | 행정안전부 주민등록 인구통계 2025년 12월 31일 | 시도별 총인구와 18세 이상 인구 사용 |


### 3.2 공식 데이터 수집 방식

`python -m scripts.fetch_official_data` 명령은 공식 원천 자료를 `data/raw/official/`에 내려받고, 분석 입력 스키마에 맞춘 CSV를 `data/official/`에 생성한다. 현재 관람 데이터는 한국프로스포츠협회 PDF 보고서의 표를 기준으로 배구/시즌/구단/경기장 컬럼을 포함해 정리한 뒤 시도 단위 분석에 활용하며, 체육시설은 실내 및 배구 가능 체육관 기준으로 필터링하고 인구 데이터는 공식 공공 통계 값을 표준 컬럼으로 정리한다.

공식 사이트가 PDF, HTML, JSON/XML OpenAPI처럼 서로 다른 형식을 제공하므로, 스크립트는 원문 다운로드와 표준 CSV 생성을 분리한다. 향후 인증키가 있는 공공데이터 OpenAPI나 안정적인 CSV 다운로드 URL이 확보되면 동일한 출력 스키마를 유지한 채 live API 변환 로직으로 교체할 수 있다.

## 4. 분석 절차

### 4.1 데이터 수집 및 정제

1. `scripts.fetch_official_data`로 공식 원천 자료를 내려받고 분석용 CSV를 생성하거나, 원천 데이터를 CSV/API 형태로 수집한다.
2. 지역명, 종목, 시즌, 구단, 경기장, 시설 유형, 실내외 구분 등 주요 컬럼을 표준화한다.
3. 결측치, 중복 행, 이상치를 점검한다.
4. 경기 수, 관람객 수, 시설 수, 인구 수를 17개 시도 단위로 집계한다.

### 4.2 파생 변수 생성

- `경기당 평균 관람객 = 관람객 수 / 경기 수`
- `인구 대비 관람률 = 관람객 수 / 총인구 * 100`
- `인구 10만 명당 배구 가능 실내체육시설 수 = 배구 가능 실내체육시설 수 / 총인구 * 100000`
- `청년·학생층 비중 = 타깃 연령대 인구 / 총인구 * 100`
- `경기 공급 밀도 = 경기 수 / 총인구 * 100000`

### 4.3 지표 설계

| 지표 | 구성 변수 | 해석 |
| --- | --- | --- |
| 관람수요지수 | 관람객 수, 경기당 평균 관람객, 인구 대비 관람률 | 현재 관람 활성화 수준 |
| 체육인프라지수 | 배구 가능 실내체육시설 수, 인구 10만 명당 배구 가능 실내체육시설 수 | 지역 스포츠 인프라 기반 |
| 인구잠재력지수 | 총인구, 청년·학생층 비중 | 신규 팬 확장 가능 인구 기반 |
| 공급공백지수 | 인구잠재력, 체육인프라, 경기 공급 밀도 역방향 지표 | 수요 기반 대비 경기 공급 부족 정도 |

모든 지표는 Min-Max Scaling을 적용하여 0~100점 기준으로 표준화한다. 공급공백지수는 경기 공급이 적을수록 높은 점수가 되도록 역방향 정규화를 적용한다.

### 4.4 최종 잠재수요 점수 산출

기본 가중치는 다음과 같이 설정한다.

| 지표 | 가중치 |
| --- | ---: |
| 관람수요지수 | 30% |
| 체육인프라지수 | 25% |
| 인구잠재력지수 | 25% |
| 공급공백지수 | 20% |

최종 점수는 다음 공식으로 계산한다.

```text
배구 관람 잠재수요 점수 = 관람수요지수 * 0.30
                      + 체육인프라지수 * 0.25
                      + 인구잠재력지수 * 0.25
                      + 공급공백지수 * 0.20
```

가중치는 분석 목적에 따라 조정 가능하며, 민감도 분석을 통해 가중치 변화에 따른 순위 변동을 검토한다.

### 4.5 군집분석 및 유형화

K-means 군집분석을 활용하여 지역을 다음 네 가지 유형으로 분류한다.

1. **기존 흥행지역**: 관람 수요와 경기 공급이 모두 높은 지역
2. **잠재 고수요지역**: 인구·인프라 기반은 높으나 경기 공급이 부족한 지역
3. **인프라 보완지역**: 인구 기반은 있으나 체육 인프라가 부족한 지역
4. **장기 육성지역**: 관람 수요, 인구 기반, 인프라가 모두 낮아 장기적 접근이 필요한 지역

군집 수는 기본적으로 4개로 설정하되, 실루엣 점수와 엘보우 방법을 활용해 적정 군집 수를 검토한다.

## 5. 결과 시각화

- 지역별 잠재수요 점수 순위표
- 시도별 관람수요지수, 체육인프라지수, 인구잠재력지수, 공급공백지수 비교 막대그래프
- Folium 기반 잠재수요 점수 단계구분도
- 군집 유형별 지도 시각화
- 공급 대비 잠재수요가 높은 지역의 우선순위 매트릭스

## 6. 기대 결과 및 활용 방안

분석 결과를 통해 현재 관람객이 많은 지역뿐 아니라 경기 공급 대비 팬 확장 가능성이 높은 지역을 도출한다. 예를 들어 체육시설과 인구 기반은 충분하지만 프로배구 경기 공급이 적은 지역은 컵대회, 올스타전, 팝업 응원전, 유소년 배구교실의 우선 후보지로 제안할 수 있다.

활용 방안은 다음과 같다.

- 프로배구 컵대회 및 이벤트 개최 후보지 선정
- 구단별 원정 마케팅 및 지역 팬덤 확장 전략 수립
- 유소년 배구교실, 학교 연계 프로그램 우선지역 선정
- 지자체 체육시설 활용 및 스포츠 관광 정책 수립
- 연고지 외 지역의 잠재 팬 대상 홍보 캠페인 설계

## 7. 한계 및 보완 방향

- 관람객 데이터가 프로배구 단일 종목으로 충분하지 않을 경우 프로스포츠 전체 관람 데이터를 보조 지표로 활용해야 한다.
- 시도 단위 분석은 시군구 단위의 세부 접근성 차이를 반영하기 어렵다.
- 실제 경기장 접근성, 교통 소요시간, 구단 선호도, 미디어 노출량 등은 추가 데이터 확보 시 보완한다.
- 최종 우선지역 선정 시 정량 점수뿐 아니라 지자체 협력 가능성, 경기장 대관 가능성, 지역 배구협회 활동 수준 등 정성 정보를 함께 검토한다.

## 8. 구현된 분석 코드

현재 `src/volleyball_demand_analysis.py`에는 실제 CSV 데이터를 입력받아 점수 산출부터 유형화, 결과 저장까지 수행하는 실행 코드가 구현되어 있다. 외부 분석 패키지 설치가 어려운 환경에서도 실행할 수 있도록 Python 표준 라이브러리만 사용한다.

구현 흐름은 다음과 같다.

1. 관람 데이터, 체육시설 데이터, 인구 데이터를 로컬 CSV 또는 HTTP(S) CSV URL로 읽는다.
2. 영문·국문 컬럼명 별칭을 자동 인식한다.
3. 지역명을 17개 시도 표준명으로 변환한다.
4. 지역별 경기 수, 관람객 수, 시설 수, 인구 수를 집계한다.
5. 파생 변수와 4개 지표를 계산한다.
6. 최종 잠재수요 점수를 산출하고 순위를 부여한다.
7. K-means 군집분석으로 지역 유형을 분류한다.
8. CSV 결과표, JSON 요약, Markdown 리포트, Folium HTML 지도를 저장한다.

실행 예시는 다음과 같다.

```bash
python -m src.volleyball_demand_analysis \
  --attendance data/official/attendance.csv \
  --facilities data/official/facilities.csv \
  --population data/official/population.csv \
  --output data/processed/region_scores.csv \
  --summary data/processed/summary.json \
  --report data/processed/report.md \
  --map data/processed/volleyball_demand_map.html
```


### 8.1 URL 기반 입력

분석 CLI의 `--attendance`, `--facilities`, `--population` 인자는 로컬 파일 경로와 HTTP(S) URL을 모두 받을 수 있다. URL을 쓰면 공식 데이터나 기관 데이터 포털의 CSV 다운로드 링크를 직접 연결해 분석할 수 있으므로 입력 데이터를 매번 저장소에 직접 넣지 않아도 된다.

다만 현재 구현은 CSV 텍스트를 직접 반환하는 URL을 전제로 한다. 인증키가 필요한 Open API, JSON, XML, 엑셀 다운로드 링크는 먼저 표준 CSV 컬럼으로 변환한 후 사용한다.

`data/official/` 입력 파일은 공식 보고서와 공공 통계 기반으로 구성되어 있으나, 관람 데이터는 2020~2022년 구단별 최근 3년 평균 관중 수를 시도별로 합산한 값이다. 최신 시즌의 시도별 관중 원자료를 확보하면 같은 CSV 형식으로 교체하여 분석을 재실행한다.
